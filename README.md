# 微信读书 Wiki

一个与微信读书同步的个人阅读知识库 — 基于 Andrej Karpathy 的 **LLM Wiki** 模式构建。

> **在摄取时编译知识，查询时只读已编译的 Wiki — 不再重复检索原始资料。**

---

## 为什么不用纯 RAG？

传统 RAG 在查询时检索原始文本片段。而本 Wiki 在同步时就已经：
- 提炼每本书的核心观点
- 建立跨书籍的概念关联
- 综合多本书对同一主题的不同视角
- 生成阅读路径和主题分析

当你读了 50 本书后，Wiki 能回答的问题深度远超简单检索。

## 三层架构

```
┌──────────────────────────────────────────┐
│  Layer 3: Schema  (CLAUDE.md)            │  ← 规则、模板、工作流
├──────────────────────────────────────────┤
│  Layer 2: Wiki    (wiki/*)               │  ← LLM 生成与维护
├──────────────────────────────────────────┤
│  Layer 1: Sources (sources/*)            │  ← 从微信读书 API 同步
└──────────────────────────────────────────┘
```

## 核心操作

| 操作 | 触发方式 | 说明 |
|------|----------|------|
| **Init** | 首次运行 | 构建目录骨架和初始页面 |
| **Sync** | "从微信读书同步" | 拉取最新数据，更新 sources 和 wiki |
| **Query** | 任何问题 | 从 wiki 页面回答，引用具体页面 |
| **Lint** | "检查 wiki" | 报告断链、孤页、缺失概念 |

## 快速开始

### 第 1 步 — 在 LLM 工具中打开项目

推荐：**CodeBuddy**、**Claude Code** 或 **Cursor**

### 第 2 步 — 同步数据

> 请读取 `CLAUDE.md` 并从微信读书同步最新数据到 wiki。

### 第 3 步 — 查询知识库

> 我读过哪些关于「自由意志」的书？它们有什么不同观点？
>
> 推荐一个关于「经济学入门」的阅读路径。
>
> 我今年的阅读偏好有什么变化？

## 推荐工具链

| 层级 | 工具 | 用途 |
|------|------|------|
| 浏览 | **Obsidian** | 原生 `[[wikilink]]` 渲染 + 图谱视图 |
| LLM 引擎 | **CodeBuddy / Claude Code** | 读写本地文件，执行同步/查询 |
| 版本控制 | **Git (私有)** | 每次同步生成 diff |
| 数据源 | **微信读书 API** | 书架、笔记、统计数据 |

---

## 数据源

本知识库通过微信读书 Agent API Gateway 获取数据：
- 书架信息、阅读进度
- 划线与笔记
- 阅读统计（时长、天数、偏好）
- 书籍详情
